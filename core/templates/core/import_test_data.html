📝 Format Requirements

  • Size values: Use S, M, L, XL, XXL, XXXL. Avoid 2XL or 3XL (they're normalized automatically).
  • Colours: Match existing designs when importing printed SKUs and orders.
  • Numbers: Use integers for quantities and inventory counts.
  • Optional fields: Leave blank if not needed (e.g., variant, tags, placement_note).
  • First row: Must be a header row with column names.
  • Encoding: CSV must be UTF-8 or UTF-8 with BOM.

🧪 Quick Test Scenarios

Common testing flows you can build:

  • Full workflow: Import designs → blank SKUs → printed SKUs → orders → test print batch and receive.
  • Missing size testing: Create orders with blank size field to test missing-size warnings.
  • Multi-variant: Import designs with variants (e.g., "Ugadi Telugu Raasi Phalalu" with "Karkataka", "Vruschika", "Thula").
  • Inventory shortage: Import low blank SKU quantities and high order demand to test shortfall detection.
  • Different statuses: Create orders at needs_printing, in_printing, ready_to_ship to test state transitions.
