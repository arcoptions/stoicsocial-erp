# Extended Abstract
## Abstract
StoicSocial ERP is an internal operations platform designed for a print-on-demand apparel business model in which demand capture, print production planning, stock movement control, and shipment readiness must be orchestrated with high consistency. The project problem addressed in this work is the operational fragmentation that typically occurs when order intake, print scheduling, receive reconciliation, and inventory correction processes are handled across disconnected spreadsheets and ad hoc communication channels. The proposed system centralizes these workflows in a Django-based architecture with auditable state transitions, role-aware pages, and deterministic service logic for stock mutation.

StoicSocial ERP was analyzed through the lens of operational reliability scenario 1. The implementation demonstrates how allocation and receiving workflows reduce execution variance while preserving visibility. The analysis was based on concrete entities such as Orders, OrderLines, PrintedSKU, PrintJob, PrintJobLine, StockMovement, and operational dashboards for inventory and finance teams. This project adopts a systems perspective where each status transition is validated against data integrity, auditability, and role-based workflow control.

H0-1: Implementing StoicSocial ERP does not significantly improve order-to-shipment flow visibility compared to spreadsheet-based tracking.

H1-1: Implementing StoicSocial ERP significantly improves order-to-shipment flow visibility through status dashboards, filters, and role-specific queues.

H0-2: Transaction-safe inventory services do not significantly reduce stock inconsistency events.

H1-2: Transaction-safe inventory services with row-level locking significantly reduce stock inconsistency events.
